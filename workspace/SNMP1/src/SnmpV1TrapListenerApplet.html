jSNMP Enterprise™ SNMPv1 Trap Listener Applet

This page displays the jSNMP Enterprise™ sample applet SnmpV1TrapListenerApplet below, which demonstrates how to use jSNMP Enterprise™ from within a browser.

The example requires a fully JDK1.1-compliant browser, and has been tested successfully using the Sun JDK AppletViewer, Netscape Communicator 4.06, and Microsoft Internet Explorer 4.01. To run the example use the following steps:

  1. Create a directory below jSNMP/examples called classes. Copy the files jSNMP/examples/SnmpV1TrapListenerApplet.java and jSNMP/lib/jSNMPEnterprise.jar into the jSNMP/examples/classes directory
  2. Take the following steps to compile the applet:
    • Launch a shell in the jSNMP/examples/classes directory.
    • Ensure that the shell's CLASSPATH environment variable contains the jSNMP/examples/classes/jSNMPEnterprise.jar, as well as the jSNMP/examples/classes directory.
    • Compile the applet as follows:
        javac SnmpV1TrapListenerApplet.java
  3. Browser-specific configuration:
    • Netscape - In order to run the examples in Netscape Communicator, a feature known as Codebase Principals must be used, which allows a codebase to act as a principal for enabling privileges. You will need to gather all classes required for the applet under the specified codebase. In this example the codebase used is the classes directory underneath jSNMP/examples. The following are detailed instructions:
      1. Ensure that no jSNMP™ classes are on your CLASSPATH environment variable. Netscape uses the CLASSPATH before any CODEBASE defined in the HTML page. Any classes loaded from the CLASSPATH can not be used to enable privileges.
    • Internet Explorer - Internet Explorer allows a local CLASSPATH to act as a principal for enabling privileges, so the only step required is to make sure the CLASSPATH is initialized correctly.
      1. Ensure that the CLASSPATH environment variable contains the jSNMP/examples/classes/jSNMPEnterprise.jar, as well as the jSNMP/examples/classes directory.
  4. Take the following steps to run the applet:
    • Start the browser, and load jSNMP/examples/SnmpV1TrapListenerApplet.html (this file).
    • To test trap handling, configure remote SNMP enabled hosts to send traps to the machine where the browser is running.
    • To filter traps by host, enterprise OID, and/or trap number, fill in the appropriate fields.
